# Icon


<BetaDocsBanner />
<UntestedBanner />

With the help of the **Icon** component, icons from integrated icon fonts can be displayed at any position. The output of the icon can be controlled using the **`_icons`** attribute and is made barrier-free using the **`_label`** attribute. By default the output is as an _`inline`_ element.

Currently the icons are supported by <kol-link _href="https://develop--kolibri-public-ui.netlify.app/#/icon/all-kolicons" _target="_blank" _label="Kolicons"></kol-link> and <kol-link _href="https://fontawesome.com" _target="_blank" _label="Font Awesome"></kol-link>.

<kol-alert _heading="Note" _type="info">It is important that the CSS file(s) of the icon font(s) are included in the host page (`index.html`).</kol-alert>

## Construction

The **Icon** component is created via the HTML tag `kol-icon`.

<IconPreview
	visibleProperties={['_label', '_icons']}
	codeCollapsable
	codeCollapsed
/>

### Icons

The property `_icons` expects a string with the class names of the icons to be shown (e.g.: `_icons="kolicon-house`).

<kol-link _href="https://develop--kolibri-public-ui.netlify.app/#/icon/all-kolicons" _target="_blank" _label="Kolicons overview"></kol-link>

## Accessibility

It is important for context-relevant graphics to be labeled.

- <kol-link _href="https://www.w3.org/WAI/ARIA/apg/practices/names-and-descriptions/" _target="_blank" _label="WAI-ARIA: Names and Descriptions"></kol-link>

### Labels

A context-relevant icon must be labeled using the `_label` property.
