# Dialog

<BetaDocsBanner />
<UntestedBanner />

The `Dialog` component provides a full-screen or focused modal overlay for delivering critical, time-sensitive information or capturing focused user input that requires the user's immediate attention. It is designed to appear on top of all other page content.

<DialogPreview componentName="dialog" />

## Usage

Use the `Dialog` component when you need to:
*   Confirm an irreversible action (e.g., "Are you sure you want to delete this record?").
*   Display important, non-dismissible alerts.
*   Collect a small, critical piece of information (e.g., a confirmation code).

### API

The `Dialog` component is controlled by props that manage its visibility, title, content, and actions.

**Props:**

| Name | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `isOpen` | `boolean` | `false` | Controls the visibility of the dialog. Must be explicitly managed by the parent component state. |
| `onClose` | `() => void` | `() => {}` | Callback function fired when the user dismisses the dialog (e.g., by clicking outside or pressing Escape). |
| `title` | `string` | `"Dialog"` | The main title displayed at the top of the modal. |
| `primaryActionLabel` | `string` | `"Confirm"` | The label for the primary action button. |
| `secondaryActionLabel` | `string` | `"Cancel"` | The label for the secondary/dismiss action button. |
| `children` | `ReactNode` | `null` | The content to be displayed inside the modal body. This is the primary area for user feedback or forms. |

### Variants

| Variant | Use Case | Notes |
| :--- | :--- | :--- |
| `confirm` | For mandatory user confirmation. | Includes both primary and secondary action buttons. |
| `alert` | For displaying information that requires acknowledgement. | Only shows a dismiss button. |
| `input` | For collecting single pieces of input. | Should wrap an input component within its `children`. |

### API Details

*   **`isOpen`**: `boolean`. Controls visibility.
*   **`onClose`**: `() => void`. The required callback for dismissal handling.
*   **`title`**: `string`. Sets the header text.
*   **`primaryActionLabel`**: `string`. Sets the main button label.
*   **`secondaryActionLabel`**: `string`. Sets the cancel button label.
*   **`children`**: `ReactNode`. Must contain the core content, form elements, or structured text.

*Note: The dialog must be controlled by its parent component's state. Always use `isOpen` and `onClose` callbacks together.*
