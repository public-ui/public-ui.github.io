## Quick start: New project

The fastest way to get started is using the command line assistant:


## Frequently asked questions

<details>
<summary><strong>Components not showing?</strong></summary>

Ensure that `register()` was called **before** rendering and the promise is resolved.

```tsx
// ‚úÖ Correct
register(DEFAULT, defineCustomElements)
  .then(() => {
    // Render app here
  });

// ‚ùå Wrong
register(DEFAULT, defineCustomElements); // Promise not awaited
// App renders immediately
```
</details>

<details>
<summary><strong>Which theme should I use?</strong></summary>

- **`@public-ui/theme-default`** ‚Äì Neutral standard theme (recommended for getting started)
- **`@public-ui/theme-ecl`** ‚Äì European Commission Library theme
- [Create your own theme](/docs/concepts/styling/theming)
</details>

<details>
<summary><strong>TypeScript errors with properties?</strong></summary>

Check:
1. `"moduleResolution": "Node"` in `tsconfig.json`
2. Framework adapter installed (e.g. `@public-ui/react-v19`)
3. Components imported from adapter (not from `@public-ui/components`)
</details>

<details>
<summary><strong>Can I use KoliBri without a build tool?</strong></summary>

Yes! Load KoliBri via CDN:

```html
<script type="module">
  import { register } from 'https://unpkg.com/@public-ui/components@latest/dist/esm/index.js';
  import { defineCustomElements } from 'https://unpkg.com/@public-ui/components@latest/dist/loader/index.js';
  import { DEFAULT } from 'https://unpkg.com/@public-ui/theme-default@latest/dist/index.mjs';

  register(DEFAULT, defineCustomElements);
</script>
```

‚ö†Ô∏è **For production:** Use fixed versions instead of `@latest`
</details>
