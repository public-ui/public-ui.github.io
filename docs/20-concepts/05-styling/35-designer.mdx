# Theme für KoliBri erstellen

> Hinweis: Der bisherige Designer steht nicht mehr zur Verfügung; Themes werden nun direkt im Code gepflegt.

Die Erstellung eines eigenen Themes erfolgt vollständig in einem Git-Repository. Das Vorgehen orientiert sich an der Dokumentation im [KERN Developer Kit](https://gitlab.opencode.de/kern-ux/kern-developer-kit/-/blob/main/CONTRIBUTING.md).

## Vorgehen

1. **Bestehendes Theme forken:** Nutzen Sie ein aktuelles Theme als Ausgangspunkt und erstellen Sie daraus ein eigenes Repository (z. B. per Fork oder `git clone`). So behalten Sie die Struktur bei, die KoliBri für das Styling erwartet.
2. **Styleguide anwenden:** Passen Sie die vorhandenen Styles, Variablen und Tokens an Ihren Styleguide an. Arbeiten Sie dabei direkt im Code des geforkten Themes, um Farben, Abstände, Typografie und Komponentenvarianten konsistent abzuleiten.
3. **Theme versionieren:** Committen Sie Ihre Anpassungen regelmäßig und halten Sie das Theme-Repository sauber dokumentiert. Damit stellen Sie sicher, dass Ihr Theme langfristig nachvollziehbar und wiederverwendbar bleibt.
4. **Theme nutzen:** Binden Sie das fertige Theme aus Ihrem Repository in Ihre Projekte ein. So können Anwendungen die angepassten Styles ohne den Designer verwenden.

## Hinweise

- Wenn Sie globale CSS-Regeln benötigen (z. B. für Zoom-Stufen, Hyphenation oder Abstände), ergänzen Sie diese direkt in Ihrem Theme.
- Für Sass-Workflows können Sie weiterhin den [Scss-Ansatz](/docs/concepts/styling/scss) verwenden, um Ihre Styles modular zu organisieren.
