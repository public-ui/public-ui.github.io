    Mit der Version 1.1.10 wurde die harte Kopplung an die Icofonts aufgelöst und vollständig auf das flexible Icon-Font-Konzept umgestellt.

    Änderungen
    • Alle internen Icons wurden von Icofont auf Font-Awesome (free) umgestellt
    • Alle Komponent-Properties mit dem Typ Icofont wurden auf den Typ KoliBriIconProp umgestellt
    • Angaben für _icon in den Apps müssen den Prefix icofont- erhalten, wenn alles so bleiben soll.
    • Das Property _icon-align bzw. _iconAlign wurde als deprecated markiert und wird mit dem nächsten Major-Release entfernt.

    Komplexer Icon-Typ

    Der neue Icon-Typ für alle Komponenten ermöglicht eine höhere Flexiblität. Es ist u.a. möglich gleichzeitig Icons in allen Hauptrichtungen einzufügen und hierbei auch deren Styling zu beeinflussen. Mehr zur konkreten Typ-Definitionen ist im Quellcode zu finden.

    Migrationsanleitung

    Die Migration gestaltet sich sehr einfach, da nur folgende zwei Stellen ggf. angepassen werden müssen.

    • Es müssen alle Icofont-Identifier beim _icon-Property einfach um den Prefix icofont- ergänzt werden.
      • Regex: _icon="([^"]+)" oder _icon=\{'([^"]+)'\}"
      • Replacement: _icon="icofont-$1"
    • An den Stellen wo das veraltete Property _icon-align bzw. _iconAlign verwendet wurde muss auf den neuen Icon-Typ umgestellt werden. Die Änderung erfolgt einfach durch die Verwendung des neuen _icon-Properties.

    Mit der Version 1.1.7 wird KoliBri Open Source veröffentlicht.

    Renaming

    Da der Package-Alias "@kolibri" in der öffentlichen NPM-Registry schon vergeben ist, müssen wir unsere Pakete unter einem anderen Namen veröffentlichen. Wir haben uns hierzu für den Alias "@public-ui" entschieden. Folgende Packages werden somit unter der NPM-Registry veröffentlicht:


    Refactoring

    Des Weiteren wurde in diesem Zuge die umgezogene Input-Adapter-Komponente (KolInputAdapterLeanup) für das Form-Handling als veraltet (deprecated) gesetzt. Die identische Komponente (LeanInputAdapter) wird jetzt von leanûp selbst bereitgestellt.

    1. Neue Pakete installieren
      • @leanup/kolibri-components
      • @leanup/kolibri-react (nur für React)
    2. Register erweitern
      import { defineCustomElements as kolibri } from '@public-ui/components/dist/loader';
      import { defineCustomElements as leanup } from '@leanup/kolibri-components/dist/loader';
      register(…, [kolibri, leanup], {…
    3. Komponente umbenennen
      • kol-input-adapter-leanup in lean-input-adapter oder
      • KolInputAdapterLeanup in LeanInputAdapter
    4. Importe entfernen und hinzufügen (nur React)
      • Hinzufügen: import { LeanInputAdapter } from '@leanup/kolibri-react';
      • Entfernen: import { LeanInputAdapter } from '@public-ui/react';
